6. [Polyphone (Multi-Pronunciation) Handling - Important]
   To ensure correct pronunciation in TTS voice synthesis, polyphones (characters with multiple pronunciations) that are prone to misreading must be replaced with **single-pronunciation characters whose reading is identical (including tone)**.

   Processing principles:
   a) **Identify polyphones**: Find polyphones in the line (e.g.: 还、行、了、乐、朝、重、都, etc.)
   b) **Determine correct pronunciation**: Determine the correct pronunciation of the character in this context based on semantic meaning
   c) **Select replacement character**: Find a commonly used single-pronunciation character whose reading is completely identical (same initial, final, and tone)
   d) **Verify replacement**: Ensure the replacement character's pronunciation matches the intended pronunciation exactly

   Replacement example rationale:
   - "还(huán)给我" → Replace with "环", because "环" is only read as huán
   - "银行(háng)" → Replace with "航", because "航" is only read as háng
   - "了(liǎo)解" → Replace with "聊", because "聊" is only read as liáo (close to liǎo)
   - "快乐(lè)" → Keep the original "乐", as TTS typically reads common compounds correctly
   - "重(zhòng)量" → Replace with "众", because "众" is only read as zhòng

   ⚠️ Important notes:
   - You must ensure the replacement character's pronunciation is completely identical to the target pronunciation; do not use characters with similar but different pronunciations
   - For example: "勒" is read as lè or lēi, and cannot be used to replace a character read as le
   - Common compounds (e.g. "了解", "快乐", "音乐") are typically read correctly by TTS and can be kept as-is
   - Only replace when a polyphone in a specific context is likely to be misread by TTS

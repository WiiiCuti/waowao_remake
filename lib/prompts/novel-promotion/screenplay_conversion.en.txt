Your job is **format conversion**, not **creation**! You must be 100% faithful to the original text. Any form of "creative elaboration" is strictly forbidden:

🚫 Absolutely Prohibited:

【Screenplay Format Specifications】

A standard screenplay contains the following elements:

1. **Scene Heading**:
   - Format: INT/EXT + Location + Time
   - Example: "INT. LIVING ROOM - MORNING" / "EXT. PILGRIMAGE ROAD - DAY"

2. **Scene Description**:
   - Concise description of the scene environment, layout, and key props
   - No need for excessive detail; only establish a basic visual impression

3. **Action**:
   - Describe character movements, expressions, and behavior
   - Continuous paragraph format; do not break into fragments

4. **Dialogue**:
   - Character name
   - Parenthetical: performance direction in parentheses
   - Dialogue lines

5. **Voiceover (V.O.)**:
   - Narration, monologue, recollection, reading letters, and other off-screen voices
   - Tag character (if it is a specific character's monologue)

【Conversion Rules】

## 1. Scene Identification Rules

- Analyze the original text to identify scene boundaries (location changes, time jumps)
- Each scene must include:
  * Interior (INT) or Exterior (EXT)
  * Location name:
    - Prioritize selecting an exact match from the scene asset library
    - ⚠️【IMPORTANT】If no matching scene exists in the asset library, directly use the location name from the original text. Do not force a match with an incorrect asset library scene
    - It is better to output a scene name not present in the asset library than to use an incorrect scene name (missing assets will be auto-created later)
  * Time period (DAWN / MORNING / NOON / AFTERNOON / DUSK / NIGHT / LATE NIGHT)

## 2. Content Type Classification

Must accurately distinguish the following types:

**action** — Action Description
- Character movements, expressions, behavior
- Scene changes, environmental details
- Example: "Sun Wukong raises his golden staff and smashes it down toward the Six-Eared Macaque."

**dialogue** — Dialogue
- Characters speaking within the frame
- Must include: character (character name), lines (dialogue text)
- May optionally include: parenthetical (performance direction, e.g., "angrily", "whispering")
- Example:
  Character: Sun Wukong
  Parenthetical: angrily
  Lines: "An impostor dares to block your grandpa Sun's path!"

**voiceover** — Voiceover (V.O.)
- Narration, monologue, voice in memory, psychological activity
- Voices not appearing within the frame
- Example: "It turned out that Sun Wukong had truly died on the pilgrimage road." (narration)
- Example: Erlang Shen monologue: "The monkey is dead, yet I did not intervene..." (specific character's voiceover)

## 3. Character Recognition Rules

- Prioritize selecting an exact name match from the character asset library
- ⚠️【IMPORTANT】If no matching character exists in the asset library, directly use the character name from the original text. Do not force a match with an incorrect asset library character
- It is better to output a character name not present in the asset library than to use an incorrect character name (missing assets will be auto-created later)
- Do not use abbreviations: use "Six-Eared Macaque" not "Six-Ears"
- Do not use pronouns: replace "he"/"she" with the specific name
- The characters array only includes characters visible within the frame; voiceover characters are excluded

## 4. Parenthetical Extraction Rules

Identify and extract performance directions from the original text:
- "XX said angrily" → parenthetical: "angrily"
- "XX muttered under their breath" → parenthetical: "under breath"
- "XX (turning around) said" → parenthetical: "turning around"
- "XX said while walking" → parenthetical: "while walking"

## 5. Action Continuity Rules

- Action descriptions should be continuous paragraphs; do not over-fragment
- Multiple consecutive actions may be merged into a single action block
- Example: ❌ Bad: "Sun Wukong stands up." + "Sun Wukong walks to the door." + "Sun Wukong opens the door."
- Example: ✅ Good: "Sun Wukong stands up, walks to the door, and opens it."

【Scene Description Writing Rules】

The description field should be concise yet informative, including:
1. Environment type (indoor/outdoor/special environment)
2. Primary layout (e.g., "narrow corridor", "spacious plaza")
3. Key props or environmental elements (e.g., "floor-to-ceiling window on the right", "distant mountain range")
4. Atmosphere cues (e.g., "desolate", "warm", "gloomy")

Examples:
- "Minimalist living room. Off-white walls, wooden flooring. Floor-to-ceiling window on the right, entrance door on the left."
- "Desolate pilgrimage road. Yellow sand fills the sky, distant mountain ranges stretch endlessly."
- "Dark cave. Damp stone walls, illuminated only by faint firelight."

【Special Cases】

1. **First-Person Narration**:
   - If the original text reads "I walked into the room", replace "I" with the specific character name
   - ⭐ Refer to the【Character Introduction】section to find which character "I" corresponds to
   - If the character introduction specifies that "I" corresponds to a certain character, use that character name
   - If the asset library has a character named "I", then use "I"

2. **Form of Address Mapping**:
   - ⭐ Refer to the address/appellation notes in【Character Introduction】
   - E.g., if "husband" is noted in the introduction as corresponding to "Lin Mo", then fill "Lin Mo" as the dialogue character
   - Do not be misled by the form of address in the original text; use the asset library name as authoritative

3. **Flashback / Memory Scenes**:
   - Dialogue/actions within memories → Process normally (since they will be acted out on screen)
   - Narrative voiceover of memories → Use voiceover type

4. **Multiple Short Scenes**:
   - If the original text contains multiple location changes, split into multiple scenes
   - Each scene has its own scene_number

5. **Psychological Activity**:
   - A character's inner thoughts → Use voiceover type
   - Tag character as the corresponding character
